# React-паттерны проекта

## Структура файлов

- Все компоненты лендинга — в `src/App.tsx` (один файл, текущий паттерн)
- Выносить в отдельный файл только если компонент > 150 строк и логически самостоятельный
- CSS-переменные и токены — только в `src/index.css` через `@theme`

## Компоненты

- Functional components, никаких классов
- Props с inline-типами для простых случаев, `interface` для сложных
- `any` — запрещён (использовать конкретные типы)

## Изображения

- Все в `public/`, путь `/filename.webp`
- Всегда `loading="lazy"` кроме hero (первый экран)
- `alt=""` для декоративных, осмысленный alt для контентных

## Scroll-анимации

```tsx
// Стандартный паттерн входа
viewport={{ once: true }}           // срабатывает один раз
viewport={{ once: true, margin: '-80px' }}  // с отступом для групп
```

- `once: true` — обязательно, чтобы не повторялось при скролле вверх
- `staggerChildren` для списков: `0.08–0.15s`

## Responsive

- Mobile-first через Tailwind breakpoints: `md:`, `lg:`
- Текст адаптируется через clamp — никаких `text-sm md:text-lg`
- Flex/grid переключение: `flex-col md:flex-row`
